# 일정 분류 기준 (SOT — generator·evaluator·golden·audit 공용)

`schedule_status` ∈ `"yes" | "pending" | "no"`. **의미 하나로** 분류한다.

## 핵심 질문 (이것만 보고 정한다)
**"사용자에게 미래에 직접 갈/할 일이 있는가? 그게 확정인가?"**

- **yes** = 확정 — 내가 그 시각에 확실히 참석/수행한다. (개인 약속, 예약·면접·진료·항공·방문서비스 등 확정 알림)
- **pending** = 가능성 — 내가 갈/할 수 있으나 아직 미정. (설명회·포럼·세미나 참석 안내, 개인 신청·신고·납부·등록·반납·검진·마감, 조인/참가 요청)
- **no** = 없음 — 자동 거래·통보·정보. (결제·적립·배송·인증, 광고·홍보, 인사·안부·잡담, 과거 회고, 남의 일정, **기업 모집 등 "내 행동 아닌" 것**)

## 절대 원칙
★ **날짜·시각은 분류 근거가 아니다 — 추출값일 뿐이다.**
- 거래·광고·통보·인사는 날짜·금액·기간·시각이 붙어 있어도 **no** ("결제예정 6/17", "적립 6/15 10:43", "~6/7 50% 할인", "월요일 아침입니다"). **메시지의 날짜를 보고 발화하지 마라.**
- 반대로 날짜·시각이 전혀 없어도 내 행동이면 **pending** ("장학금 신청해야 해", "건강검진 예약하자"). 없는 날짜를 지어내지 마라(date=null).

★ **순서**: ① yes/pending/no 분류 → ② yes·pending이면 date/time을 표면형 그대로 추출(없으면 null). no면 `events: []`. (confidence 없음 — status가 확실성을 표현.)

## 경계 (헷갈리는 것)
- **모집·공고·안내**: 내가 참석·신청할 행동이면 **pending** / 단순 정보·홍보·**기업 모집(내 행동 아님)**이면 **no**.
- **개인 마감/할일**(신고·납부·신청·등록·반납·검진) = **pending**. 단 **완료 통보**("자동납부 완료"·"자동이체 등록 완료")·광고·취소·과거는 **no**.
- **배송**: "방문 수령" + 일시(내가 가야 함) = yes / 단순 도착 통보 = no.

## 멀티턴(대화) — 내용이 아니라 "확정 도달 여부"로
- **yes** = 특정 날짜·시각에 **합의**, 마지막이 수락/재확인 ("네 화요일 13시 봬요", "내일 뵙겠습니다"). 하나의 약속 = 한 건, 최종 합의값 추출.
- **pending** = 구체 시각 **제안으로 끝남** ("화요일 13시 어때요?", "6/4 9시 가능하세요?") — 아직 미수락.
- **no** = **거절·취소·유보·모호** ("다른 일정 있어 다음에", "회의 취소", "조만간 보자").

## 대조 예시 (표면형 같아도 라벨 다름 — 토큰 의존 차단)
| 메시지 | 라벨 | 이유 |
|---|---|---|
| 내일 14시 강남역에서 봐요 | **yes** | 확정 약속, 내가 감 |
| [병원] 6/3 10:30 진료 예약 | **yes** | 확정 예약 |
| 전문가 설명회 1/28 14시 안내 | **pending** | 참석 가능, 미확정 |
| 국가장학금 신청 마감 6/19 | **pending** | 내 행동, 미확정 |
| 장학금 신청해야 하는데 | **pending** | 날짜 없어도 내 행동 |
| 기업 2차모집 공고 6/19 | **no** | 기업 모집 = 내 행동 아님 |
| 카카오페이 적립 6/15 10:43 | **no** | 거래 통보(날짜 있어도) |
| Buddy Pass 결제예정 6/17 | **no** | 자동 결제 |
| 오후에 연락드릴게요 | **no** | 영업 로지스틱스 |
| 어제 세미나 잘 마쳤습니다 | **no** | 과거 회고 |
| 화요일 13시 어때요?(제안) | **pending** | 미확정 제안 |
| 다른 일정 있어 다음에 | **no** | 거절 |
